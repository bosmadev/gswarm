plugin for Claude Code. Here's what the video covers based on related sources:

## The Ralph Loop Concept

The "Ralph Wiggum Technique" is a simple bash loop that runs Claude continuously:

```bash
while :; do cat PROMPT.md | npx --yes @sourcegraph/amp ; done
```

Created by Geoffrey Huntley in July 2025, it went viral by late 2025. The core
philosophy is: "state lives in files, not conversation. Each iteration
reconstructs reality from the filesystem - not from a polluted transcript of
what went wrong. Progress persists. Failures evaporate."

## Criticisms of the Official Plugin

The official Anthropic Ralph Loop plugin has been criticized for several reasons:

1. **Technical Issues:**
   - "It dies in cryptic ways unless you have --dangerously-skip-permissions"
   - Installs hooks in hard-to-find locations
   - Uses opaque markdown files for state tracking
   - Causes complete breakage if you delete the markdown file before properly
     disabling the plugin

2. **Architectural Problems:**
   - Uses a stop hook that blocks exit and re-feeds the prompt in the same session
   - The transcript grows and context fills up
   - Failed attempts pollute future iterations
   - This defeats the original purpose of fresh context per iteration

3. **Conceptual Misalignment:**
   The core criticism is philosophical: "it misses the key point of ralph which
   is not 'run forever' but 'carve off small bits of work into independent
   context windows.'"

## The Original Vision vs Official Implementation

The original Ralph approach had file I/O as state instead of transcript. The
agent should re-anchor from sources of truth on every pass. Anthropic's own
long-context guidance says exactly this: agents must re-anchor from sources
of truth to prevent drift.

## Alternative Recommendation

Rather than using the official plugin, many developers prefer simple bash loops:
"I'll keep my 5-line bash loops" - allowing for fresh context windows on each
iteration while maintaining state in the filesystem.
